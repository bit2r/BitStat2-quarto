# BitStat2 Live 홈 UI 재설계 (2026-07-06)

**결론**: BitStat2-quarto(코드학습 트랙)의 홈 `index.qmd`를 원점에서 재설계했다.
링크 한 줄짜리 스텁을 **커리큘럼 맵 랜딩**으로 교체 — 히어로(가치제안+CTA) →
학습 방식 3요소 → 7챕터·23토픽 전체 지도(라이브/준비 중 상태 표기) → 안내 노트.
콘텐츠 구조는 BitStat2 모노레포(`pages/01_data`~`07_theory`)에서 가져왔다.
설계 시각화: [2026-07-06_live_홈UI_재설계.svg](../../images/2026-07-06_live_홈UI_재설계.svg) (Tufte 스타일).

**최종(v2)**: 상단 navbar 제거 — 홈 자체가 내비게이션(CTA·커리큘럼 맵)이므로 중복
크롬이었다. 콘텐츠 페이지는 상단에 `← BitStat2 Live 홈` 링크 하나만 둔다.
배포: **https://r2bit.com/BitStat2-quarto/** (bit2r/BitStat2-quarto, GitHub Pages `main:/docs`).

## 1. 배경과 목표

- 분리 직후 BitStat2-quarto 홈은 제목 + 중심극한정리 링크 1개뿐 — 사이트의
  정체("코드를 고쳐 실행하며 배우는 트랙")와 전체 규모가 드러나지 않았다.
- 목표: ① 첫 화면에서 **무엇이 다른 사이트인지**(편집형 셀·반응형·자동 채점) 선언,
  ② 전체 커리큘럼 지도를 처음부터 공개하되 **현재 라이브 1개**를 정직하게 표기,
  ③ 유일한 라이브 토픽으로 직결되는 1차 CTA — 막다른 탐색 없음.

## 3. 화면 구조 (위→아래)

0. **상단 메뉴 없음** — `_quarto.yml`에서 navbar 제거. 홈이 곧 내비게이션이고,
   콘텐츠 페이지(clt 등)는 본문 최상단 `← BitStat2 Live 홈` 링크로 복귀한다.
1. **히어로** — eyebrow(BITSTAT2 LIVE) / 제목 "코드를 직접 고쳐 실행하며 배우는 통계" /
   가치제안 한 문장(편집·실행 + 반응형 + 자동 채점) / 근거 메타 한 줄(webR·설치 불필요·짝 사이트) /
   CTA 2개(실선: 중심극한정리 실습, 고스트: #curriculum 앵커) / **진행 헤어라인**(2px, 1/23 ≈ 4.3%).
2. **학습 방식 3요소** — 편집형 R 셀 · 반응형 시각화(OJS×webR) · 자동 채점(check).
   상하 헤어라인으로만 구획.
3. **커리큘럼 맵** — 챕터 카드 7개 그리드(`auto-fill minmax(310px,1fr)`).
   카드: 번호(연회색)·제목·부제·토픽 리스트. 상태는 **색+기호+텍스트 삼중 병기**:
   ●(액센트)+링크+LIVE 배지 vs ○(연회색)+일반 텍스트. 색 단독 의존 금지.
4. **푸터 노트** — webR 첫 로드 소요 안내 + 콘텐츠 원본(BitStat2) 링크.

## 4. 디자인 토큰

Tufte 원칙(잉크 최소·헤어라인·직접 라벨) + 현대적 카드 그리드의 절충.

| 토큰 | 값 | 용도 |
|---|---|---|
| ink | `#111827` | 본문·제목 |
| muted / faint | `#6b7280` / `#9ca3af` | 보조 텍스트 / 메타·준비 중 |
| hairline | `#e5e7eb` | 구획선·카드 테두리(1px) |
| accent | `#2563eb` (hover `#1d4ed8`) | CTA·라이브 상태·링크 — **유일한 유채색** |

- 액센트는 dataviz 팔레트 검증 통과(lightness band·chroma·대비 ≥3:1, light 서피스).
- 그림자 없음, 반경 8–10px, 구획은 여백과 헤어라인이 대신한다.

## 5. 구현

- 파일: `index.qmd`(raw HTML 블록) + `assets/home.css`(`.bs-` 접두사 스코프,
  `_quarto.yml` 아닌 **문서 front matter `css:`**로 홈에만 적용).
- front matter: `engine: knitr`(필수) · `pagetitle`(title 대신 — 타이틀 블록 억제) ·
  `page-layout: full` + 컨테이너 max-width 1080px 자체 관리 · `toc: false`.
- webR 셀 없음 → 홈은 런타임 로드 없이 즉시 뜬다.

### 구현 함정 2건 (재발 방지)

1. **Quarto가 본문 첫 `<h1>`을 문서 제목으로 끌어올린다**(title 미지정 시).
   히어로 제목이 페이지 상단으로 분리됨 → `<h1>` 대신 `<div class="bs-h1">` 사용.
2. **raw HTML(````{=html}`) 안에서는 `.qmd` 링크가 재작성되지 않는다**
   (마크다운 링크만 변환됨) → `pages/.../index.html`로 직접 지정.

## 6. 검증

- `quarto render` 정상, 실브라우저(1546px) 확인: 히어로/3요소/7카드 그리드/노트
  모두 의도 레이아웃, LIVE 링크 → clt 페이지 정상 연결.
- 반응형: 그리드 `auto-fill`·히어로 `clamp()` — 좁은 폭에서 1열로 붕괴(별도 브레이크포인트 불요).

## 7. 남은 일

- 토픽 이식 시 홈의 해당 `<li>`에 `bs-live` 클래스+링크+배지 추가, 진행 헤어라인
  width(%)와 "n개 라이브" 캡션 갱신 (수동 — 토픽 수가 적어 자동화 이득 없음).
- navbar는 최종 제거됨(v2). 새 토픽 이식 시 콘텐츠 페이지 상단 `← 홈` 링크 패턴을 복제할 것.
  라이브 토픽이 많아져 페이지 간 이동 수요가 생기면 그때 사이드바/navbar 재검토.
- 다크 모드는 cosmo 라이트 단일 — 요구 발생 시 토큰만 반전.
